## Critical CSS Bug Fix (Safari/WebKit)

**Problem:** MetalForge.io showed blank screen in Safari with error:
`TypeError: Failed to set an indexed property [0] on 'CSSStyleDeclaration'`

**Root Cause:** `react-native-web` tries `element.style[0] = value` which fails in Safari/WebKit.

**Solution:** Runtime Proxy wrapper in `fixCssStyles.js` that intercepts numeric property sets:
- Uses WeakMap keyed on `realStyle` object for reference equality
- Must be imported FIRST in App.js (before any other imports)
- Metro bundler ignores `patch-package` patches on `node_modules` - runtime patching required

**File:** `packages/frontend/fixCssStyles.js`

## Search Bar Native Input Fix

**Problem:** After CSS fix, react-native-web TextInput stopped accepting text input.

**Root Cause:** The Proxy wrapper interferes with TextInput's internal style handling.

**Solution:** Replace TextInput with native HTML `<input>` element:
- Use `Platform.OS === 'web'` check to render native input on web
- Keep TextInput fallback for native mobile platforms
- Bypasses react-native-web entirely on web

**Mobile attributes added:**
- `inputMode="text"` - triggers mobile text keyboard
- `enterKeyHint="search"` - shows search button on keyboard  
- `minHeight: 44` - proper touch target size
- `WebkitAppearance: 'none'` - removes iOS default styling
- `touchAction: 'manipulation'` - improves touch responsiveness

**Location:** SearchBar component at line ~1591 in `packages/frontend/App.js`

## Key Architectural Learnings

1. **Metro bundler ignores node_modules patches** - Can't use patch-package for react-native-web fixes
2. **Runtime patching is the only reliable fix** - Must patch before first render
3. **Native HTML elements bypass react-native-web issues** - When RNW components break, use native equivalents with Platform check
4. **9 other TextInputs in App.js** may need similar native input treatment (lines 4281, 4876, 6401, 6983, 7667, 11451, 11916, 12090, 15461)
